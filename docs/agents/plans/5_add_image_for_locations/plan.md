# Plan: Add Image for Locations

## Overview
Add image support to the home location (and establish the pattern for future locations) by storing image assets under `personal_marketing/assets/images/locations/` and importing them into the location definition files.

## Context
The Edwin engine already supports a location `image` property. The home location currently has no image. Images must work both in the Vite dev server and in the final production build — Vite handles this automatically when images are imported as ES modules.

## Implementation Steps

### Step 1 — Add the home image asset
Place the home location image at:
```
personal_marketing/assets/images/locations/home.png
```

### Step 2 — Import and assign the image in the location definition
In `personal_marketing/lib/locations.js`, import the image as an ES module and pass it to the home `Location` constructor:

```javascript
import homeImage from '../assets/images/locations/home.png';

new Location({
  // ...existing properties...
  image: homeImage,
});
```

## Files to Change
- `personal_marketing/assets/images/locations/home.png` — new image asset for the home location
- `personal_marketing/lib/locations.js` — import the image and assign it to the home location definition

## Notes
- Vite resolves image imports at build time, so no extra configuration is needed for production builds
- This establishes the pattern for adding images to other locations in the future
- The image file itself must be provided (it is a binary asset, not generated by code)
